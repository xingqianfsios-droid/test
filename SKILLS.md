# ♟️ Xiangqi Game Logic Skills

## 1. 走法验证逻辑 (Move Validation)
当 Claude 需要实现棋子移动逻辑时，应参考以下准则：
- **帅/将**: 只能在九宫格内移动，不能准对。
- **士**: 只能在九宫格内斜走。
- **象**: 田字走法，需检查“象眼”是否被塞。
- **马**: 日字走法，需检查“马腿”是否被别。
- **炮**: 移动同车，翻山需中间恰好有一个棋子。

## 2. 状态表示 (State Representation)
- 棋盘状态应使用一维数组 `List<Piece?> (90个元素)` 或二维数组表示。
- 使用 `GetxController` 的 `update()` 或 `Rx` 变量触发棋盘重绘。

## 3. UI 实现技巧
- **棋盘绘制**: 使用 `CustomPaint` 绘制复古木质线条。
- **棋子**: 使用 `Stack` + `AnimatedPositioned` 实现丝滑的走棋动画。
- **反转棋盘**: 对于黑方视角，计算坐标时使用 `(8-x, 9-y)`。

## 4. 常见算法
- **将军检查**: 模拟移动后，检查对方是否可以直接吃掉己方主帅。
- **胜负判定**: 找不到任何合法的下一步走法即为“绝杀”或“困毙”。

## 5. 常见问题
- **棋子被吃掉UI问题**: 一方的棋子如果在另一个的范围被吃掉的时候UI刷新异常。
